# Case Study: Рефакторинг 92 листов ГПР — Победа над "призрачным" кэшем ГП5

###  Executive Summary: Итог битвы за стерильность данных
*   **Проблема:** Выгрузка из ГРАНД-Сметы на 92 листах (каша в столбцах). На масштабе вскрылся "спящий" баг: с 7-го листа и далее в рабочей зоне появлялись данные от сметы "ГП 5" из скрытого подвала шаблона.
*   **Решение Симбиоза:** Внедрены "tripwires" (датчики-взрыватели) в макрос. Найден скрытый XML-шлейф. Написан бронированный VBA-монолит с "Операцией 0" (хирургическое отсечение подвала) до расчета.
*   **Результат:** 92 листа очищены за <5 секунд. Точность 100%.

---

###  Анатомия расследования: Вскрытие ловушки
При масштабировании макроса на 92 листа (используя логику из), код "взрывался" на 7-м листе.
1.  **Гипотезы (Flash Fill, кэш `ActiveSheet`, перенос данных) — не подтвердились.**
2.  **Момент истины:** Внедрены визуальные датчики. Взрыв на датчике №0 до начала расчетов.
3.  **Причина:** При удалении пустых строк во 2-й фазе (см.), скрытый мертвый текст `ГП5` из "подвала" (ниже строк с данными) вываливался в рабочую область.

---

###  Инженерные монолиты: Продакшн-код (VBA)

Разработан скрипт с предварительной чисткой, обеспечивающий 100% стерильность данных.

```vba
' --- ОПЕРАЦИЯ 0: ХИРУРГИЧЕСКОЕ ОТСЕЧЕНИЕ ГРЯЗНОГО ПОДВАЛА ЛИСТА ---
firstBlankRow = 0
For i = 2 To lastRow + 1
    If Trim(ws.Cells(i, "D").Value) = "" And Trim(ws.Cells(i, "E").Value) = "" Then
        firstBlankRow = i: Exit For
    End If
Next i
If firstBlankRow >= 2 Then
    On Error Resume Next
    ws.Rows(firstBlankRow & ":" & ws.Rows.Count).ClearContents
    ws.Rows(firstBlankRow & ":" & ws.Rows.Count).Delete
    On Error GoTo 0
End If
' ... далее следуют фазы реверсивного переноса и нормализации из
```

*(Полный код для 1 и 92 листов, использующий логику из `fnal.md`, находится в исходном репозитории).*

###  Философия Контура: Всему своя норма
Глобальный макрос (92 листа) подтвердил техническую возможность, но Оператор отдал приоритет **локальному применению**. Это гарантирует:
*   Визуальный контроль каждой итерации.
*   Исключение пропуска аномалий в закрытом движке смет.

**Вывод:** Истинная автоматизация — это не слепой цикл, а контроль над ситуацией.
