# Кейс: Оператор нового века (Дирижер процессов) 

### Вводные
**Контекст:** На руках у инженера сложная выгрузка ГПР (92 листа). Задача — вытащить данные по трудозатратам и привязать их к технологическим работам для переноса в другую мастер-таблицу. Ручной разбор занял бы около **12–16 часов** монотонной работы. 

**Решение:** Вместо того чтобы тратить два рабочих дня на механическое копирование, Оператор разворачивает симбиоз с ИИ, мгновенно превращая хаотичную структуру в автоматический конвейер. Ошибки исключены, время решения задачи — **5 минут**.

---

### Эталон взаимодействия: Как дирижирует Оператор

Ключевой фактор скорости — четкое ТЗ и жесткие логические границы, заданные человеком в один заход. Оператор не просит абстрактно «сделай красиво», а выдает готовую механику алгоритма:

> **Прямой запрос Оператора:**
> *«вот структура - смотри в графу ед. изм и выполняем поиск, как только доходим до чел-час (внимание - не чел-час(м)), мы берем значение из равой соседней графы (F), вырезаем его и переносим в другую ячейку - графа H., строка - тут внимание - дойдя до чел-час и захвотя значение из его соседа, мы возвращаемся вверх - идем до предыдущего чел-часа, но не заходим в него, а останавливаемся перед ним и заносим вырезанное значение в графу H соответстующей строки., ниже структура (в строке 1 - у нас шапка) - нам нужен скрипт, который будет работать аналогично на други листах - создаем конвейер»*

**Почему это ТЗ идеального Дирижера:**
1. **Заданы жесткие маркеры:** Выделен целевой триггер (`чел-час`) и сразу отсечен ложный шум (`чел-час(м)`).
2. **Прописана уникальная реверсивная логика:** Алгоритм «сканируем вниз ➔ забираем значение ➔ возвращаемся вверх до границы предыдущего блока» полностью закрывает проблему отсутствия жестких связей в таблице.
3. **Задан масштаб:** Сразу обозначено, что структура листов сквозная, и нужен скрипт-конвейер для масштабирования на всю книгу.

### Техническое решение конвейера (VBA макрос)

На основе алгоритма Оператора ИИ сгенерировал безотказный инструмент, который работает прямо внутри Excel без установки сторонних программ.

```vba
Sub MoveManHours()
    Dim ws As Worksheet
    Dim lastRow As Long
    Dim i As Long, j As Long
    Dim valToMove As Variant
    Dim prevCheckRow As Long
    
    ' Работаем с активным листом
    Set ws = ActiveSheet
    
    ' Находим последнюю заполненную строку по колонке D
    lastRow = ws.Cells(ws.Rows.Count, "D").End(xlUp).Row
    
    ' Переменная для отслеживания строки предыдущего чел-час (для старта это строка 1 - шапка)
    prevCheckRow = 1
    
    ' Идем циклом сверху вниз, начиная со строки 2 (под шапкой)
    For i = 2 To lastRow
        
        ' Проверяем строгое соответствие "чел-час" (исключая "чел-час(м)")
        If Trim(ws.Cells(i, "E").Value) = "чел-час" Then
            
            ' 1. Забираем значение из колонки F
            valToMove = ws.Cells(i, "F").Value
            
            ' 2. Очищаем ячейку в колонке F
            ws.Cells(i, "F").Value = ""
            
            ' 3. Логика подъема вверх: определяем целевую строку
            ' Если предыдущего чел-час не было (prevCheckRow = 1), пишем в строку 2
            ' Иначе — в строку сразу перед текущим чел-час
            Dim targetRow As Long
            If prevCheckRow = 1 Then
                targetRow = 2
            Else
                targetRow = i - 1
            End If
            
            ' 4. Переносим значение в колонку H целевой строки
            ws.Cells(targetRow, "H").Value = valToMove
            
            ' Обновляем координату последнего найденного чел-час
            prevCheckRow = i
            
        End If
    Next i
    
    MsgBox "Готово! Данные перенесены.", vbInformation, "Конвейер"
End Sub

```
