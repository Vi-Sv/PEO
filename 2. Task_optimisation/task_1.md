# Human-AI Co-Pilot Case: Скоростной реинжиниринг сметных данных ГПР

### Суть инцидента
Необходимо было трансформировать 92 уникальных листа графиков производства работ (ГПР) для миграции в целевую базу данных. Ключевая рутина — ручной захват нормативов `чел-час` и их корректная привязка к родительским технологическим этапам. 


---

### Архитектура симбиоза: Роль Оператора-Дирижера

Ключевой фактор скорости в данном кейсе — отказ от парадигмы «ИИ, сделай за меня». Оператор выступил в роли архитектора и дирижера процесса, задав жесткие границы и математическую логику:

1. **Идентификация паттерна:** Оператор мгновенно декомпозировал хаотичную визуальную структуру таблицы (где связь «работа — ресурс» держалась только на отступах) в строгий алгоритм для машины.
2. **Проектирование реверсивной логики:** Оператор сформулировал уникальный принцип «сканирование вниз → реверсивный подъем до границы блока данных → фиксация в строке родительской работы». Это исключило необходимость писать сложные тяжелые парсеры.
3. **Фильтрация шума:** Оператор сразу выставил жесткое ограничение на маркеры-абстракции (срезать `чел-час`, строго игнорировать машинные затраты `чел-час(м)`).
4. **Выбор среды:** Вместо развертывания инфраструктуры Python, оператор выбрал нативный VBA, встроив конвейер прямо в текущую сессию Excel.

(черновой запрос: "вот структура - смотри в графу ед. изм и выполняем поиск, как только доходим до чел-час (внимание - не чел-час(м)), мы берем значение из равой соседней графы (F), вырезаем его и переносим в другую ячейку - графа H., строка - тут внимание - дойдя до чел-час и захвотя значение из его соседа, мы возвращаемся вверх - идем до предыдущего чел-часа, но не заходим в него, а останавливаемся перед ним и заносим вырезанное значение в графу H соответстующей строки., ниже структура (в строке 1 - у нас шапка) - нам нужен скрипт, который будет работать аналогично на други листах - создаем конвейер. вот структура листа (она будет аналогичной на всех последуюи)")

### Техническое воплощение алгоритма (VBA)

```vba
Sub MoveManHours()
    Dim ws As Worksheet
    Dim lastRow As Long
    Dim i As Long, j As Long
    Dim valToMove As Variant
    Dim prevCheckRow As Long
    
    ' Работаем с активным листом
    Set ws = ActiveSheet
    
    ' Находим последнюю заполненную строку по колонке D
    lastRow = ws.Cells(ws.Rows.Count, "D").End(xlUp).Row
    
    ' Переменная для отслеживания строки предыдущего чел-час (для старта это строка 1 - шапка)
    prevCheckRow = 1
    
    ' Идем циклом сверху вниз, начиная со строки 2 (под шапкой)
    For i = 2 To lastRow
        
        ' Проверяем строгое соответствие "чел-час" (исключая "чел-час(м)")
        If Trim(ws.Cells(i, "E").Value) = "чел-час" Then
            
            ' 1. Забираем значение из колонки F
            valToMove = ws.Cells(i, "F").Value
            
            ' 2. Очищаем ячейку в колонке F
            ws.Cells(i, "F").Value = ""
            
            ' 3. Логика подъема вверх: определяем целевую строку
            ' Если предыдущего чел-час не было (prevCheckRow = 1), пишем в строку 2
            ' Иначе — в строку сразу перед текущим чел-час
            Dim targetRow As Long
            If prevCheckRow = 1 Then
                targetRow = 2
            Else
                targetRow = i - 1
            End If
            
            ' 4. Переносим значение в колонку H целевой строки
            ws.Cells(targetRow, "H").Value = valToMove
            
            ' Обновляем координату последнего найденного чел-час
            prevCheckRow = i
            
        End If
    Next i
    
    MsgBox "Готово! Данные перенесены.", vbInformation, "Конвейер"
End Sub

```

### Таймлайн и Метрики

* **Этап постановки ТЗ и калибровки логики:** 3 минуты.
* **Этап генерации и проверки кода:** 2 минуты.
* **Время работы конвейера на 1 лист:** < 1 секунды.
* **Итоговая экономия времени:** ~15 чистых человеко-часов.
* **Точность переноса данных:** 100% (человеческий фактор полностью исключен на этапе проектирования ТЗ).

**Вывод:** Эффективность ИИ определяется не мощностью его языковой модели, а точностью разметки границ и ТЗ со стороны человека. Когда оператор умеет мыслить алгоритмически, ИИ превращается в безотказное продолжение его рук.
