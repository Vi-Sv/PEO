# Профессиональная самопрезентация: Архитектура ИИ-контура и автоматизация в ПЭО

## Введение: Роль Архитектора и ИИ-оператора
Мой подход к автоматизации — это не ручное написание кода, а проектирование логики, управление нейросетями и жесткий аудит результатов. Я декомпозирую бизнес-задачи, формулирую строгие ТЗ для ИИ и управляю вычислительными процессами. Мой опыт включает три этапа автоматизации ПЭО:

---

## Этап 1. Milestone_task: Аудит данных и унификация
*   **Задача:** Обработка >3 100 строк сметных выгрузок с хаотичными единицами измерения (`м2`, `10 м2` и т.д.).
*   **Решение:** Разработка математической логики и ТЗ для ИИ на создание VBA-макроса, приводящего данные к эталонам (`м2`, `м3`, `шт`) с пересчетом норм ГЭСН.
*   **Ключевая ценность:** Внедрение тройного контура проверки (через `IF-ELSE`) для исключения ошибок, что обеспечило 100% точность.

---

## Этап 2. Task_optimisation: Реверс-инжиниринг ГПР
*   **Задача:** Обработка «вертикальной каши» из 92 листов графиков производства работ.
*   **Решение:** Проектирование алгоритма для создания VBA-конвейера. Реализован инвертированный поиск (связка ресурсов с работами) и замена статических данных на живые формулы (`Объем × Трудоемкость`).
*   **Ключевая ценность:** Автоматизация 14-18 часов ручного труда; обработка всей книги за 5 секунд с локальным контролем.

---

## Этап 3. Task_GASN: Инфраструктурный парсинг данных
*   **Задача:** Привязка 633 позиций к нормам ГЭСН, преодоление лимитов контекста чатов ИИ.
*   **Решение:** Использование GitHub для передачи данных ИИ, разработка VBA-скрипта с логикой аппроксимации (`1.25 ч/ч`) для автоматического заполнения пустых зон.
*   **Ключевая ценность:** Мгновенное (2 сек) создание готовой базы данных при локальном выполнении кода.

---

## Итоговые компетенции
1.  **Экономия времени:** Создание промышленных инструментов автоматизации вместо ручного программирования.
2.  **Надежность:** Тотальный контроль математики и аудит результатов ИИ.
3.  **Управление ИИ:** Использование ограничений нейросетей для выстраивания эффективных цепочек данных.
