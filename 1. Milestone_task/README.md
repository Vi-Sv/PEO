## Суть проблемы
При обработке выгрузки сметных/нормативных данных объемом **971 строка** мы столкнулись с классической проблемой «грязных» данных: разнобой в единицах измерения площадей (`м2`, `10 м2`, `100 м2`, `1000 м2`). Для автоматизации расчетов ч/ч требовалось жестко привести все строки к единому знаменателю — `м2`, пропорционально пересчитав исходные объемы и нормы расхода.

## Системный тупик (Интерфейсный барьер)

## Нестандартное решение (Связка «Оператор + ИИ»)
Вместо бесперспективной борьбы с текстовыми фильтрами, мы перестроили архитектуру взаимодействия и вынесли вычисления за пределы чата. Мы создали автономный конвейер:

1. **Инфраструктура (Оператор):** Развертывание сырого массива данных на внешнем репозитории GitHub (`task1.txt`) для сквозного доступа.
2. **Инструмент автоматизации (ИИ):** Разработка точечного отказоустойчивого макроса VBA под внутренний движок Excel для мгновенного локального пересчета.
3. **Локальное выполнение (Оператор):** Накатывание макроса напрямую в рабочую книгу Excel. Приведение базы к эталону за 1 секунду в обход ограничений сети.
4. **Автономный аудит (ИИ):** Слепое построчное сопоставление исходного (`task1.txt`) и обработанного (`task2.txt`) файлов через внутренний Python-интерпретатор для 100% верификации точности.

## Закрывающий инструмент (VBA Макрос) /код сгенерирован ИИ, протестирован оператором

```vba
' Данный макрос локально выравнивает единицы измерения к м2, 
' корректируя объемы (× коэффициент) и нормы расхода (/ коэффициент).
Sub ConvertUnitsToM2_StrictSafe()
    Dim ws As Worksheet
    Dim lastRow As Long
    Dim i As Long
    Dim unitStr As String
    Dim coef As Double
    
    ' Работаем строго с текущим активным листом Excel
    Set ws = ActiveSheet
    
    ' Находим последнюю заполненную строку по Столбцу 1 (А)
    lastRow = ws.Cells(ws.Rows.Count, "A").End(xlUp).Row
    
    ' Выключаем обновление экрана для моментального расчета
    Application.ScreenUpdating = False
    
    ' Начинаем со 2-й строки, так как 1-я — это шапка таблицы
    For i = 2 To lastRow
        ' Важнейший сброс: для каждой новой строки коэффициент равен 1
        coef = 1
        
        ' Берем текст из Столбца 6 (Ед.изм.), убираем пробелы и переводим в строчные буквы
        unitStr = Trim(ws.Cells(i, 6).Value)
        unitStr = Replace(unitStr, " ", "")
        unitStr = LCase(unitStr)
        
        ' Жесткая проверка на абсолютное совпадение текстовых строк
        If unitStr = "10м2" Then
            coef = 10
        ElseIf unitStr = "100м2" Or unitStr = "100м" Then
            coef = 100
        ElseIf unitStr = "1000м2" Then
            coef = 1000
        End If
        
        ' Точечный пересчет запускается ТОЛЬКО если найден укрупненный коэффициент
        If coef > 1 Then
            ' Столбец 3: Объем (Исходный) -> Умножаем на коэффициент
            If IsNumeric(ws.Cells(i, 3).Value) And ws.Cells(i, 3).Value <> "" Then
                ws.Cells(i, 3).Value = ws.Cells(i, 3).Value * coef
            End If
            
            ' Столбец 4: Вып. до 04.2026 -> Умножаем на коэффициент
            If IsNumeric(ws.Cells(i, 4).Value) And ws.Cells(i, 4).Value <> "" Then
                ws.Cells(i, 4).Value = ws.Cells(i, 4).Value * coef
            End If
            
            ' Столбец 5: План на объем Begin -> Умножаем на коэффициент
            If IsNumeric(ws.Cells(i, 5).Value) And ws.Cells(i, 5).Value <> "" Then
                ws.Cells(i, 5).Value = ws.Cells(i, 5).Value * coef
            End If
            
            ' Столбец 6: Переименовываем укрупненную единицу в чистые м2
            ws.Cells(i, 6).Value = "м2"
            
            ' Столбец 7: Норма расхода ч/ч -> Жестко делим на коэффициент
            If IsNumeric(ws.Cells(i, 7).Value) And ws.Cells(i, 7).Value <> "" Then
                ws.Cells(i, 7).Value = ws.Cells(i, 7).Value / coef
            End If
        End If
    Next i
    
    ' Возвращаем обновление экрана в исходное состояние
    Application.ScreenUpdating = True
    
    MsgBox "Контроль завершен! Укрупненные единицы приведены к м2. Строки с обычными м2 полностью защищены от изменений.", vbInformation, "Успех"
End Sub


```

## Личная верификация результатов (Железобетонный аудит)

Для гарантированного и ручного аудита, локально на ПК в Excel на двух чистых листах со значениями (Лист 1 — исходник `task1`, Лист 2 — результат после макроса `task2`) была создана колонка с вычислением суммарных трудозатрат по формуле:

$$\text{Трудозатраты} = \text{Объем} \times \text{Норма}$$

После протягивания формулы по всем 971 строкам, через функцию «Автосумма» были получены два абсолютно идентичных числа итоговых человеко-часов на обоих листах. 

Это фундаментально свидетельствует о правильном решении: макрос отработал без сдвигов, пропусков и округлений, сохранив общий баланс трудоемкости проекта, но полностью ликвидировав разнобой в единицах измерения.

 **Тотальный текстовый аудит (Контроль структурного сдвига):**
   Чтобы полностью исключить иллюзию слепой веры и проверить неизменность позиций, которые вообще не должны были участвовать в расчете, Оператор провел перекрестную текстовую верификацию. Сравнив текстовые ячейки, не учавствующие в работе при помощи формулы жесткого сравнения:
   
   ```excel
   =ЕСЛИ(Лист1!B2=B2; "ИСТИНА"; "ЛОЖЬ")
   ```
   
   Формула была протянута по всей базе до 971 строки. Через автофильтр был выполнен сквозной аудит: **на протяжении всего массива получен статус «ИСТИНА»**. Это физически доказывает, что макрос сохранил аутентичный порядок строк, не перемешал позиции и не допустил смещения сметных наименований.

 **Динамический перекрестный аудит чисел с условиями (Глубокий QA-контроль):**
   Чтобы окончательно исключить иллюзию слепой веры, Оператор инициировал обратную проверку объемов с жестким ветвлением логики (`IF-ELSE`). На листе «После» - он же "лист 2" в ячейку `I2` была внедрена комплексная формула пересчета:
   
   ```excel
   =ЕСЛИ('До'!F2="10 м2"; ЕСЛИ('До'!E2*10=E2; "ИСТИНА"; "ЛОЖЬ"); ЕСЛИ('До'!F2="100 м2"; ЕСЛИ('До'!E2*100=E2; "ИСТИНА"; "ЛОЖЬ"); ЕСЛИ('До'!F2="1000 м2"; ЕСЛИ('До'!E2*1000=E2; "ИСТИНА"; "ЛОЖЬ"); ЕСЛИ('До'!E2=E2; "ИСТИНА"; "ЛОЖЬ"))))
   ```
   
   **В ходе сквозного протягивания формулы по 971 строке была обнаружена единичная аномалия — маркер «ЛОЖЬ»**. 
   
   Оператор применил автофильтр, мгновенно локализовал дефектную строку и провел расследование. Причина оказалась в «грязном вводе» исходных данных: в проверяемой строке единица измерения была записана как `100м2` (слитно, без пробела), из-за чего формула проверки пропустила условие, в то время как наш строгий макрос VBA успешно распознал и пересчитал эту строку. 
   
   После ручной корректировки синтаксиса в формуле контроля, **весь массив на 100% окрасился в статус «ИСТИНА»**. Этот этап лично подтвердил абсолютную точность вычислений и доказал силу сквозного аналитического контроля.

## Архитектурная модель Оператора (Формула Симбиоза)

Ключевой фактор успеха — Оператор не вел себя как пассивный пользователь, ожидающий готового ответа, а выступил в роли **Системного Архитектора**. Была выстроена жесткая трехуровневая модель управления потенциалом ИИ:

1. **Декомпозиция и лимитирование («Шаг в шаг»):** Оператор сразу пресек попытки ИИ выдать весь массив данных целиком. Был установлен жесткий шаг итераций (сначала 100, затем срез до 70 и 30 строк) для контроля точности математики.
2. **Внешнее контурирование (Инфраструктурный маневр):** Как только обнаружился интерфейсный барьер платформы, Оператор мгновенно перестроил среду. Вместо бесперспективного «скармливания» кусков таблиц в чат, данные были вынесены на GitHub, что дало ИИ прямой сквозной доступ к сырому массиву и освободило контекстное окно от перегрузки.
3. **Смена парадигмы выдачи (От текста к инструменту):** Когда текстовые фильтры заблокировали передачу очищенных строк, Оператор скорректировал задачу: *«не выдавай мне данные, выдай мне инструмент для их локального контроля»*. Это перевело ИИ из режима «генератора текста» в режим «инженерного соразработчика».

Именно этот жесткий менеджмент и понимание ограничений систем позволили превратить текстового ассистента в промышленный вычислительный инструмент.
